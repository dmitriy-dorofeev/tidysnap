# Безопасность

## Защитные механизмы

### 1. Тестовый режим по умолчанию (Dry Run)

При первом запуске `dry_run` установлен в `true`. Утилита показывает, какие файлы будут удалены, но **не удаляет** их. Это позволяет убедиться в корректности настроек перед реальным удалением.

### 2. Поиск только по расширениям

Сканирование выполняется строго по расширению файла (`.png`, `.mov` и т.д.), а **не по имени**. Это исключает ложные срабатывания на файлы с похожими именами (например, `report.png.txt` не будет найден как изображение).

### 3. Запрет на корневые системные папки

Нельзя выбрать в качестве целевой директории:
- `/`
- `/System`
- `/Users`
- `~` (домашняя папка)

### 4. Предупреждение о важных папках

Если выбрана одна из следующих папок, появляется предупреждение:
- `Desktop`
- `Downloads`
- `Documents`

Пользователь должен явно подтвердить осознание риска (`warning_ack`).

### 5. Логирование всех операций

Каждая операция удаления (или dry-run) записывается в лог с временной меткой, путём и размером файла. Это позволяет отследить, что было удалено.

## Рекомендации

- **Всегда начинайте с Dry Run** — убедитесь, что сканирование находит только нужные файлы.
- **Проверяйте логи** после каждой очистки (`[l]` в TUI или `~/Library/Logs/tidysnap/cleanup.log`).
- **Не отключайте Dry Run** до полного понимания работы утилиты.
- **Создавайте резервные копии** важных данных перед первой реальной очисткой.
